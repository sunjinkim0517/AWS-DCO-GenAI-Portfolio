# CRC Error·Link Down 교육용 Incident 분석

*본 분석은 가상의 시나리오를 바탕으로 작성된 교육용 샘플 데이터 분석 자료입니다.*

- 차시: 10차시
- 작성자: 교육생
- 사용 파일: sample_dco_log.txt
- 사용 자료: 교육용 샘플 데이터

## 1. 분석 대상

- 날짜: 2026-07-03
- 샘플 장비명: SAMPLE_TOR_SW_01
- 인터페이스: Gi0/1
- Ticket ID: EDU-TKT-2026-0003
- 분석 시간: 03:05~03:30

## 2. 사건 전 상태

- [추정] Gi0/1 인터페이스는 정상적으로 통신이 이루어지고 있는 가동 상태였을 것으로 보입니다.
- [추정] 다만, 03:05 이전부터 물리적 손상이나 이물질 유입 등으로 인해 보이지 않는 미세한 패킷 전송 오류(CRC error)가 서서히 축적되고 있었을 가능성이 있습니다.

- [사실] 장애 발생 직후 Ticket(EDU-TKT-2026-0003)이 오픈되었고, 12분에 Onsite Cabling Team으로 escalated 되었습니다.
- [사실] 패치 코드 교체와 커넥터 정리 후 Link Up이 기록되었습니다.
- [사실] 03:30에 Gi0/1의 CRC Error가 0으로 기록되었습니다.

## 5. 가능한 원인

### 가능성 1
- **가능한 원인**: LC-LC multi-mode patch cord 케이블의 물리적 노화 또는 꺾임 손상
- [추정] 케이블 또는 연결 상태가 사건에 관련되었을 가능성이 있습니다.
- [추정] Onsite Cabling Team에서 LC-LC multi-mode patch cord 케이블을 새것으로 교체한 후 즉시 Link Up 및 통신이 정상화된 점으로 보아 케이블 결함이 주요 원인이었을 가능성이 높습니다.
- [미확인] 교체 이전 수거된 패치 코드의 피복 손상 여부나 실제 꺾임 상태는 로그만으로 확인할 수 없습니다.

### 가능성 2
- **가능한 원인**: Port connectors 접속 단면의 먼지 또는 이물질 유입으로 인한 신호 감쇄(Optical attenuation)
- [추정] 포트나 인터페이스 상태도 함께 관련되었을 가능성이 있습니다.
- [추정] 케이블 교체 시 port connectors를 청소(cleaned)한 후 에러가 사라진 점으로 보아 광단자 접속면의 오염이 신호 품질 저하를 유발했을 수 있습니다.
- [미확인] 청소하기 전 port connectors에 이물질이 실제로 얼마나 묻어 있었는지는 로그에 기록되지 않았습니다.

### 가능성 3
- **가능한 원인**: 트랜시버 모듈(SFP/GBIC) 접촉 불량 또는 진동으로 인한 일시적인 탈착 현상
- [추정] 케이블을 분리하고 다시 연결하는 가공 작업(Maintenance) 과정에서 단자 간의 접촉면이 교정 및 밀착되었을 가능성이 있습니다.
- [미확인] 당시 SFP 트랜시버 자체의 기계적 고장이나 수신 광량 상태 등은 현재 로그만으로 판정할 수 없습니다.

## 6. 현재 로그만으로 확인할 수 없는 내용

- [미확인] 현재 로그만으로 최초 원인을 완전히 확정할 수 없습니다.
- [미확인] CRC error가 급증하기 전 Gi0/1 포트의 수신 광량(Rx Power) 등 실시간 DOM 수치 정보는 알 수 없습니다.
- [미확인] 패치 코드 손상이 랙 내부 조작 작업이나 기계적 흔들림 등의 외부적 요인에 의해 발생했는지 여부는 확인할 수 없습니다.
- [미확인] 스위치 장비와 연동되어 통신이 중단된 서버 측 NIC(네트워크 카드) 및 드라이버의 실시간 상태 정보는 기록에 없습니다.

## 7. 추가로 확인하면 좋은 정보

1. **DOM (Digital Optical Monitoring) 수치**: [추정] 인터페이스의 송수신 광파워(Tx/Rx Power) 로그를 파악하여 물리 장치의 신호 강도 추이를 대조 분석합니다.
2. **랙 현장 작업 이력 (Rack Activity Log)**: [추정] 사건 시간 전후로 데이터센터 내 해당 랙(Rack) 주변에서 인접 장비 이동이나 케이블링 작업이 행해졌는지 기록을 조사합니다.
3. **상대 서버의 시스템 로그 (OS Messages / dmesg)**: [추정] 스위치 포트가 DOWN된 시점에 상대 서버 OS 커널 및 NIC 드라이버 단에서 수집된 링크 상태를 비교합니다.
4. **회수 케이블 실측 보고서**: [추정] 철거한 LC-LC multi-mode patch cord를 파이버 테스트기에 측정하여 물리적인 결함 및 감쇄율(Insertion loss) 수치를 파악합니다.

- **사건 중**: [사실] 03:05에 5분 내 CRC error 수치가 154까지 도달해 WARNING 알람이 발생한 뒤, 03:06에 상태가 완전히 DOWN으로 변경되어 서버와 연결이 유실되었습니다. [사실] 직후 EDU-TKT-2026-0003 티켓이 자동 오픈되고 Onsite Cabling Team으로 위임되었습니다.
- **사건 후**: [사실] 03:25에 LC-LC multi-mode patch cord 케이블 교체와 port connectors 청소가 수행된 후, 03:26에 Link Up 상태로 복구되어 03:30에는 CRC error가 0인 매우 건강한 상태(Healthy)가 완벽하게 유지되었습니다.
