# [EDU] 교육용 랙 점검 및 검증 체크리스트

> [!IMPORTANT]
> **교육용 가상 데이터 안내**
> 본 문서는 실제 AWS 내부 절차, 실제 장비, 실제 티켓, 실제 고객 정보를 포함하지 않습니다. 
> 모든 예시는 `SAMPLE`, `EDU` 접두어가 붙은 교육용 가상 데이터이며, 학습 목적으로만 활용됩니다.

---

## 1. 개요
본 체크리스트는 현장 하드웨어의 실물 물리 점검 절차가 아닙니다. 모니터링 시스템에서 생성된 **가상의 샘플 로그 및 티켓 정보**를 분석하고 문서화할 때 기록해야 하는 확인 항목들을 템플릿화한 교육용 자료입니다.

---

## 2. 티켓 및 로그 정보 분석 체크리스트

- [ ] **1. 발생 시간 확인**
  - 이벤트가 감지된 정확한 타임스탬프를 기록했는가? (예시: `2026-07-07 05:42:11 UTC`)
- [ ] **2. 샘플 장비명 확인**
  - 로그에 기재된 장비 식별자가 `SAMPLE_` 또는 `EDU_` 접두어를 사용하는 가상 장비명인가? (예시: `SAMPLE_TOR_SW_01`)
- [ ] **3. 반복 여부 확인**
  - 링크 다운(Link Down) 이전에 CRC 에러가 지속해서 증가하는 반복 패턴(Recurrence)이 존재하는가?
- [ ] **4. 티켓 상태 확인**
  - 분석 완료 후 가상 티켓 시스템의 상태를 업데이트했는가? (Open -> Assigned -> Resolved)
- [ ] **5. 원인 단정 금지 원칙 준수**
  - 로그 분석 결과만을 근거로 장애의 물리적 원인(예: "케이블 불량 확정")을 속단하여 서술하지 않았는가?
  - *올바른 작성 예시*: "CRC 에러 누적에 따른 물리 계층 불량 가능성이 의심되므로 현장 케이블 및 트랜시버 재장착(Reseat) 검토 권장"
- [ ] **6. 보안 준수 여부**
  - 문서 내에 실제 IP 주소, 장비 시리얼 넘버, 실무 토큰 등이 유출되지 않고 더미값 처리되었는가?

---

## 3. 보고서 초안 작성 메모 (Report Draft Memo)

*분석 내용을 기반으로 에스컬레이션 또는 최종 보고서 작성 시 참고할 임시 메모 영역입니다.*

- **이슈 요약**: `SAMPLE_TOR_SW_01` 장비의 특정 포트(`Ethernet 1/1`)에서 CRC 에러 증가 후 링크 다운 상태 발생.
- **예상되는 잠재 원인 (가설)**:
  - 광케이블(Fiber Cable) 커넥터 오염 또는 접촉 불량
  - SFP+ 트랜시버(Transceiver)의 불량 또는 안착(Seat) 상태 불량
  - 연결 대상 이웃 장비(Neighbor Device) 포트 단의 하드웨어 이슈
- **권장 조치 제안 (교육용)**:
  - 현장 엔지니어를 통한 물리 케이블 탈착 후 재장착(Reseat) 수행
  - SFP+ 트랜시버 교체 및 루프백 테스트 (모든 작업은 가상 장비 시뮬레이션으로 수행)
